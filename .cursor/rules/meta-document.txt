# TDA Course Project: Topological Data Analysis of GMM-VAE Latent Spaces

## Overview

This project implements and analyzes **Gaussian Mixture Model Variational Autoencoders (GMM-VAE)** with a focus on applying **Topological Data Analysis (TDA)** to understand the structure of learned latent representations. The project includes:

- **Model Training**: GMM-VAE models with CNN and MLP architectures
- **Dataset Support**: MNIST, CIFAR-10, and synthetic concentric triangles
- **TDA Analysis**: Persistence diagrams, barcodes, and topological features
- **Visualization**: UMAP projections, 3D latent space plots, and reconstruction comparisons

## Usage

### Training Models

Train a GMM-VAE model using a configuration file:

```bash
cd TDA-Proj
python Training.py --config cfgs/GMMCNNMNIST.json
```

Configuration files specify:
- Model architecture (CNN or MLP)
- Dataset (MNIST, CIFAR-10, or concentric triangles)
- Hyperparameters (embedding dimension, number of classes, etc.)
- Training parameters (epochs, batch size, optimizer, etc.)

### Generating Embeddings

Generate embeddings from trained models:

```bash
cd TDA-Proj
python GenerateEmbeddings.py --config cfgs/GMMCNNMNIST.json --model_path ckpts/model.pth
```

### Visualizing Reconstructions

Compare original and reconstructed images:

```bash
python visualize_reconstruction.py model_data/model_gmm_cnn_mnist_0.pth --num_images 8 --dataset mnist
```

### 3D Latent Space Visualization

Visualize latent representations in 3D:

```bash
python visualize_3d_ae.py
```

### TDA Analysis

#### Generating Model Data (General)

Generate ModelData for all datasets and embeddings (raw, VAE, PCA, UMAP):

```bash
# Process a single config
python generate_model_data.py --config TDA-Proj/cfgs/GMMCNNMNIST.json

# Process all configs in TDA-Proj/cfgs/
python generate_model_data.py --all

# With custom batch size and max batches
python generate_model_data.py --all --batch_size 2000 --max_batches 5
```

This creates pickled ModelData files in `model_data/` containing:
- Raw flattened data
- VAE latent embeddings
- PCA embeddings
- UMAP projections (raw, PCA, VAE)

#### Generating Persistence Diagrams and Barcodes (General)

Generate persistence diagrams and barcodes for all embeddings:

```bash
# Process a single ModelData file
python generate_tda_diagrams.py --model_data model_data/model_gmm_cnn_mnist_0.pkl

# Process all ModelData files
python generate_tda_diagrams.py --all

# With custom parameters
python generate_tda_diagrams.py --all --max_samples 500 --maxdim 2
```

This generates:
- Persistence barcode plots in `images/barcodes/`
- Persistence diagram plots in `images/pers_diags/`

For each embedding type: raw, vae, pca, umap_raw, umap_pca, umap_vae

#### Legacy TDA Analysis (Dataset-Specific)

**Note**: The following scripts are dataset-specific (hardcoded for MNIST). Use `generate_model_data.py` and `generate_tda_diagrams.py` for general use.

```bash
python tda_analysis_psdiag.py  # Legacy: MNIST-specific persistence diagrams
python tda_analysis_umap_plots.py  # Legacy: MNIST-specific UMAP plots
```

## Key Components

### Models

**GMMVAE_CNN**: CNN-based GMM-VAE for image datasets (MNIST, CIFAR-10)
- Encoder: Convolutional layers with residual blocks
- Decoder: Transposed convolutions
- Prior: Learnable Gaussian Mixture Model in latent space

**GMMVAE**: MLP-based GMM-VAE for simpler datasets
- Encoder/Decoder: Fully connected layers
- Suitable for low-dimensional synthetic data

### Datasets

- **MNIST**: 28×28 grayscale digit images (10 classes)
- **CIFAR-10**: 32×32 color images (10 classes)
- **Concentric Triangles**: Synthetic 2D dataset with topological structure

### TDA Tools

- **Ripser**: Computes persistence diagrams from point clouds
- **Persistence Diagrams**: Visualize topological features (holes, voids, etc.)
- **Barcodes**: Alternative visualization of persistence intervals
- **UMAP**: Dimensionality reduction for visualization

## Configuration Files

Configuration files in `TDA-Proj/cfgs/` specify:

```json
{
  "model": "GMMVAE_CNN",
  "dataset": "mnist",
  "embedding_dim": 32,
  "num_classes": 10,
  "conv_channels": [32, 64, 128],
  "conv_strides": [2, 2, 2],
  "epochs": 50,
  "batch_size": 128,
  "optimizer": {"name": "Adam", "lr": 0.001},
  "criterion": "mse"
}
```

## Key Scripts

### General TDA Analysis Scripts (Recommended)

- **`generate_model_data.py`**: General script to generate ModelData for all datasets (MNIST, CIFAR, ConcentricTriangles). Generates embeddings for raw data, VAE embeddings, PCA embeddings, and UMAP projections. Uses config files from `TDA-Proj/cfgs/` to determine dataset and model settings.

- **`generate_tda_diagrams.py`**: General script to generate persistence diagrams and barcodes for all embedding types. Processes ModelData files generated by `generate_model_data.py` and creates visualizations for raw, VAE, PCA, and UMAP embeddings.

### Training and Model Management

- **`Training.py`**: Main training script with epoch management, loss computation, and checkpointing
- **`load_model.py`**: Utilities for loading trained models and setting up data loaders
- **`GenerateEmbeddings.py`**: Generate embeddings from trained models and save as .npz files

### Legacy TDA Analysis Scripts (Dataset-Specific)

- **`distances.py`**: Legacy script for computing distances and processing model data (hardcoded for MNIST)
- **`tda_analysis_psdiag.py`**: Legacy script for generating persistence diagrams and barcodes (hardcoded for MNIST)
- **`tda_analysis_umap_plots.py`**: Legacy script for creating UMAP visualizations (hardcoded for MNIST)

**Note**: The legacy scripts are dataset-specific. Use `generate_model_data.py` and `generate_tda_diagrams.py` for general, dataset-agnostic analysis.

## Notes

- **General TDA Analysis**: Use `generate_model_data.py` and `generate_tda_diagrams.py` for dataset-agnostic analysis of MNIST, CIFAR-10, and ConcentricTriangles datasets
- **Dataset Loaders**: 
  - `TrainingDatasets.py`: Used during training (no embeddings)
  - `ProjectDatasets.py`: Used for analysis with precomputed embeddings (CIFAR/MNIST) or without (Triangles)
  - `ProjectDatasets.py` triangles dataloader matches `TrainingDatasets.py` defaults (num_samples=10000)
- **Embedding Types**: The general scripts generate embeddings for:
  - Raw data (flattened)
  - VAE latent embeddings (from model.encode())
  - PCA embeddings (matching VAE embedding dimension)
  - UMAP projections (2D) for raw, PCA, and VAE embeddings
- Models are run for MNIST and CIFAR-10; concentric triangles models are experimental
- Random seeds are fixed (seed=42 for UMAP, seed=69 for training) for reproducibility
- The project uses beta-VAE formulation with configurable KL weight
